本模块由两个并列的 Feature 目录组成，均遵循 Flutter `features/<feature>/<feature>_screen.dart` + 可选 `widgets/` 子目录的组织约定：
- `environment/`：`environment_screen.dart` 作为入口 `ConsumerWidget`，通过 Riverpod 同时 watch `connectionStateProvider` 与 `envStreamProvider`，在连接态下用 `MetricCard` 网格展示温度、湿度、气压、UV、海拔五个指标，并在底部嵌入 `EnvLineChart`；未连接时渲染 `_NotConnectedPlaceholder`。
- `environment/widgets/env_line_chart.dart`：独立的 `ConsumerStatefulWidget`，内部维护固定长度（maxPoints=60）的环形缓冲区，监听 `envStreamProvider` 增量追加 `EnvSnapshot`，使用 `fl_chart` 绘制带渐变填充的温度曲线，X 轴以相对秒数 `-60s..0s` 标注。
- `history/history_screen.dart`：当前为占位 Scaffold，提示后续接入 Drift 存储后再启用。
依赖方向单向向下：Screen → shared providers (`sensor_provider.dart`, `ble_provider.dart`) / shared widgets (`metric_card.dart`) / core ble interface / app theme，不反向依赖上层 app 或共享状态管理。
