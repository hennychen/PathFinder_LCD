本模块是 Flutter 侧的 BLE 连接 UI 层，按功能拆分为三个文件：
- `connection_screen.dart`：独立全屏页面（Scaffold + AppBar），通过 `ref.watch(connectionStateProvider)` 驱动 `disconnected/scanning/connecting/connected/failed` 五种状态分支渲染；扫描结果通过 `ble.devicesStream`（StreamBuilder）实时刷新。
- `ble_connection_sheet.dart`：作为 ModalBottomSheet 嵌入到 Dashboard AppBar 右上角芯片中打开，复用同一套状态机与 BleServiceInterface，但额外在 `initState` 自动请求 Android 运行时权限并触发扫描。
- `widgets/device_tile.dart`：纯展示 Widget，仅接收 name/rssi/onTap 参数，被两个父组件共同消费。

依赖方向单向指向共享层：UI 通过 Riverpod 的 `connectionStateProvider` 读取状态，通过 `bleServiceProvider` 获取 `BleServiceInterface` 抽象，再根据运行时类型是否为 `ReactiveBleService` 决定走真实 BLE 流还是 Mock 回退路径。模块不持有业务逻辑，只负责状态→视图映射。
