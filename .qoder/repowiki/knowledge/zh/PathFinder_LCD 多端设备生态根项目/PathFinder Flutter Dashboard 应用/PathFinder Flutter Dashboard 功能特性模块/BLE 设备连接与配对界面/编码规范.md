- 所有需要 BLE 能力的 StatefulWidget 使用 `ConsumerStatefulWidget` + `ConsumerState`，通过 `ref.watch` 订阅 `connectionStateProvider`、`ref.read(bleServiceProvider)` 获取服务实例。
- 对底层 BLE 实现采用“接口 + 运行时类型检查”模式：先断言 `ble is ReactiveBleService` 再访问 `devicesStream`，否则回退到静态占位 UI，避免在 Provider 注入阶段强耦合具体实现。
- 连接状态统一用 `BleConnectionState` 枚举配合 `when` 或 `switch` 做视图分支，每个分支对应一个独立的 `_buildXxxView` / 私有 StatelessWidget，保持单一职责。
- 涉及蓝牙操作的入口方法（如 `_startScan`、`_autoScan`）在执行 `ble.startScan()` 前统一调用 `[Permission.bluetoothScan, Permission.bluetoothConnect, Permission.location].request()` 申请 Android 运行时权限。
