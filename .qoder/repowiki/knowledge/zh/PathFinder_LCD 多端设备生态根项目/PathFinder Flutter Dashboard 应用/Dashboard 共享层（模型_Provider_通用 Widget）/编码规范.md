- 所有 BLE 数据模型均提供两个工厂方法：`fromBle(Uint8List)` 负责字节序校验与字段缩放（如 `/100.0`、`/10.0`），`mock(...)` 提供可配置的开发期假数据。
- Riverpod Provider 遵循“单一来源”原则：`bleServiceProvider` 作为唯一 `BleServiceInterface` 实例，其他 Provider 通过 `ref.watch(bleServiceProvider)` 复用，避免重复创建底层连接。
- 传感器 StreamProvider 采用“watch BLE + map + DAO.insert”的链式写法，在返回给 UI 之前完成一次本地落盘，保证历史查询与实时视图的数据一致性。
- UI 组件优先使用 `ConsumerWidget` 配合 `ref.watch` 订阅 Provider，而非手动 `setState`，使状态更新最小粒度化。
