本模块按职责分为三个子目录，构成 Dashboard 应用层的共享基础：
- `models/`：纯 Dart 不可变数据类（`EnvSnapshot`、`ImuSnapshot`、`EmoteInfo`、`MotionEvent`、`EmoteRules`、`EnvSnapshot`），每个类同时暴露 `fromBle(Uint8List)` 工厂方法将 BLE C2/C3/C4 特征字节的 Little-Endian 二进制帧解码为领域对象，并提供 `mock(...)` 工厂用于开发期无设备调试；`EmoteTrigger` 枚举集中维护触发码到中文标签的映射。
- `providers/`：基于 Riverpod 的轻量状态桥接层。`ble_provider.dart` 通过单例 `bleServiceProvider` 持有 `ReactiveBleService` 实例并在 `ref.onDispose` 中释放，再派生 `connectionStateProvider` 流式暴露连接状态；`sensor_provider.dart` 定义 `envStreamProvider` / `motionStreamProvider` / `emoteStreamProvider` 三个 StreamProvider，内部 watch BLE 服务与对应 DAO（`envDaoProvider`、`eventDaoProvider`、`emoteDaoProvider`），在数据下行时同步写入本地存储后再透传给 UI，实现“实时显示 + 持久化”的双写管线。
- `widgets/`：无业务耦合的展示型组件，包括带呼吸/旋转动画的 `BleStatusChip`（置于 AppBar actions 区域，点击弹出 `BleConnectionSheet`）、胶囊风格的 `MetricCard`、数值过渡动画的 `AnimatedCounter` 以及 `StatusIndicator`。
依赖方向严格单向：widgets → providers → models → core/ble 接口；本模块不直接依赖任何 feature 页面，仅被 `lib/app/app.dart` 等上层组合使用。
