- BLE 服务统一通过 `BleServiceInterface` 抽象暴露，真实实现 `ReactiveBleService` 与测试替身 `MockBleService` 并行存在，调用方只依赖接口。
- 所有对外状态变更均通过 `StreamController.broadcast()` 推送，并在 getter 中先 `yield` 当前内存状态再转发控制器流，避免首次订阅空数据。
- 数据库访问采用 Drift 的 Table class + DAO class 分层，DAO 构造函数接收 `AppDatabase` 实例并通过 `ref.watch(databaseProvider)` 在 Provider 中组装。
- CSV 导出函数保持无副作用纯函数风格：入参为 `AppDatabase`，内部自行查询、拼接 `StringBuffer`、写入 Documents 目录并返回文件路径。
