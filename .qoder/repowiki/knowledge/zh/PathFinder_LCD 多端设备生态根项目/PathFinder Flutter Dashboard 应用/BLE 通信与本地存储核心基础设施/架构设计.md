模块按横向能力划分为两个子目录，彼此无依赖：

- `ble/`：以 `BleServiceInterface`（含 `BleConnectionState` 枚举）作为唯一契约，上层通过 Riverpod Provider 注入实现；`ReactiveBleService` 基于 `flutter_reactive_ble` 完成真实扫描/连接/特征值订阅并广播 `EnvSnapshot`/`ImuSnapshot`/`EmoteInfo` 流，`MockBleService` 用 `Timer.periodic` 生成模拟数据供开发调试，二者均实现同一接口。Wi-Fi 配网逻辑下沉到独立的 `BleWifiWriter`，避免污染主服务。
- `storage/`：Drift ORM 层由 `database.dart` 中的 `AppDatabase`（`@DriftDatabase` + `LazyDatabase` 指向应用文档目录下的 `pathfinder_dashboard.sqlite`）驱动，`tables.dart` 声明 `EnvRecords`/`EventRecords`/`EmoteRecords` 三张表，对应 `dao_env.dart`/`dao_event.dart`/`dao_emote.dart` 三个 DAO（`.g.dart` 为 build_runner 生成）。`storage_providers.dart` 暴露 `databaseProvider` 与三个 DAO Provider，遵循 `ref.watch(databaseProvider)` 构造 DAO 的单向依赖方向。`csv_export.dart` 是纯函数式导出工具，仅依赖 `AppDatabase` 读取后写入 Documents 目录。

依赖方向：UI/feature → ble interface / storage providers → 具体实现或 Drift 底层，本模块不反向依赖任何 UI 代码。
