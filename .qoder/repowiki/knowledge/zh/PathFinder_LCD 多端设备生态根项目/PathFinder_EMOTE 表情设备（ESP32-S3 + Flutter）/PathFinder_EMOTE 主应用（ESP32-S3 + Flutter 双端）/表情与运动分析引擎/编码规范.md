- 每个源文件顶部用块注释说明设计理念与算法流程，并在函数前以分隔线划分逻辑区域。
- 所有可配置阈值（滤波系数、迟滞/防抖帧数、事件优先级、EAF 帧延迟等）集中定义为 `#define` 常量，避免魔法数字散落。
- 全局状态统一封装在 `static struct s_state { ... }` 中，并通过 `motion_engine_init` / `emote_engine_init` 等初始化函数清零，API 只暴露指针参数读写。
- 事件/规则采用“候选数组 + 优先级表”模式：先收集所有满足条件的候选，再遍历 `s_priority[]` 选最高者，便于新增事件只需改表。
- 对外 API 返回 `esp_err_t` 并使用 `ESP_LOGI/W/E(TAG, ...)` 记录关键路径，TAG 为短字符串常量。
