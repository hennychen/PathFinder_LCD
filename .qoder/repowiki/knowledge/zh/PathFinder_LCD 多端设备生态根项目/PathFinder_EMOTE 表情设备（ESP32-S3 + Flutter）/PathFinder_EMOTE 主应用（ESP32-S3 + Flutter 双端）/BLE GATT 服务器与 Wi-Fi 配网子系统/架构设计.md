本模块由三个并列的 C 文件组成，分别实现两类通信通道：
- `ble_gatt_server.c`：NimBLE Host 任务 + GAP/GATT 服务注册。定义单一自定义 Service `0xFE00`，内含四个 16-bit UUID 特征值：C2(Env, 0xFE02)、C3(Motion, 0xFE03)、C4(Emote, 0xFE04) 均为 READ+NOTIFY，C5(WiFi, 0xFE05) 为 WRITE+NOTIFY；通过全局句柄表 + 订阅位图跟踪连接/CCCD 状态，数据推送统一走 `put_u16le/put_u32le` 小端组装后 `ble_hs_mbuf_from_flat` + `ble_gattc_notify_custom`。
- `wifi_config_manager.c`：APSTA 状态机，按 NVS 中是否存在 `wifi_cred.ssid/pass` 决定启动 STA 直连还是 AP+Web Portal 配网；使用 FreeRTOS EventGroup 等待 `WIFI_EVENT_STA_GOT_IP` / `WIFI_EVENT_STA_DISCONNECTED`，失败重试上限 5 次后擦除凭据并重启进入配网。
- `web_portal.c`：ESP-IDF httpd + 自建 DNS 劫持 Task（UDP 53 将任意域名解析到 192.168.4.1），提供 `/`、`/api/scan`、`/api/connect`、`/api/status` 以及 `generate_204/hotspot-detect.html/ncsi.txt` 等 captive portal 探测路径，HTML 以 `const char[]` 驻留 Flash。
依赖方向：`wifi_config_manager.c` ↔ `web_portal.c` 双向调用（管理器启停 portal，portal 回调管理器保存凭据）；`ble_gatt_server.c` 通过 `ble_wifi_write_cb_t` 函数指针暴露 C5 Write 回调给上层，三者均仅依赖 ESP-IDF/NimBLE/cJSON 标准库，无跨层头文件耦合。
