模块以 `rgb_lcd_example_main.c` 为 IDF 入口 `app_main()`，按初始化顺序串联各子系统：先调用 `led_ws2812_init` 初始化 WS2812 灯带，再调用 `wifi_sta_init` 启动 Wi-Fi STA；随后通过 `esp_lcd_new_rgb_panel` 配置 480×480 16bit RGB 面板（帧缓冲位于 PSRAM，支持双缓冲/防撕裂二选一），注册 `example_lvgl_flush_cb` 作为 LVGL flush 回调并通过 VSYNC 事件回调实现可选的 tearing 避免同步；使用 `esp_timer` 产生 2ms LVGL tick，创建独立的 LVGL 任务 (`example_lvgl_port_task`)，所有 LVGL API 调用均通过递归互斥量 `lvgl_mux` 保护。

UI 层由两个并列入口组成：`ui/lvgl_demo_ui.c` 实现环形菜单（10 个图标 + 状态标签 + 动画切换），`ui/lvgl_gif_demo.c` 实现 GIF 动图加载与播放；两者都通过 `LV_IMG_DECLARE` / `LV_FONT_DECLARE` 引用 `images/` 与 `fonts/` 下由 UI 工具生成的 C 资源，这些资源在 `CMakeLists.txt` 中显式列入 SRCS。

外设驱动采用独立子目录封装：`lcd_ws2812/` 基于 RMT encoder 暴露 `ws2812_init/write/deinit` 句柄式 API；`wifi/` 封装 ESP-WIFI + ESP-EAP + SmartConfig 并仅暴露 `wifi_sta_init` / `get_wifi_signal_strength`；`LCD.c/h` 负责 LCD 硬件时序初始化。依赖方向单向收敛至 `main/rgb_lcd_example_main.c`，组件间通过头文件声明的函数/全局句柄交互，无交叉包含。
